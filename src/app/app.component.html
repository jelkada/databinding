Data binding in Angular:

  • Data binding refers to the idea of data synchronization between the model and the view.
  • When the data changes its value, the UI elements that are bound to the data will also change automatically.
  • Angular handles data binding by synchronizing the state of the view/UI with the data in the component.
  • Data-binding can be one-way, where a change in the model state affects the view, or two-way, where a change from the view affects the model.
  • Types of bindings are supported by Angular are: string interpolation, property binding, event binding and two-way binding.

Usage:

  • Import NgModule and FormsModule in app.module.ts
    import { NgModule } from '@angular/core';
    import { FormsModule } from '@angular/forms';
  • Declare the FormsModule in the imports section of the @NgModule decorator
    imports: [ FormsModule ]
  • Use double curly brackets to bind one-way using string interpolation. Ex: {{ city }}
  • Use square brackets to bind to property. Ex: [style.color]="color"
  • Use the directive [(ngModel)] for two-way data binding with a UI input element.
  • Square brackets means "evaluate" or get the value from the view, where as the parenthesis means "execute" an event, or update the conmponent from the view.
  • The square brackets [] indicate that the direction of the data is from the component to the view/UI. The parenthesis () indicate that some event will take place and the view will update the data in the the component.
  • The syntax of the two-way binding directive [(ngModel)] is referred to as "banana-in-the-box", which is a reminder that the box/square brackets are on the outside, and the expression with the parenthesis/banana is inside the square brackets.
  • To bind to a DOM event, surround the DOM event name in parentheses and assign a quoted function to it.
